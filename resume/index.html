🚀 项目经历

1. SaaS 供应链系统 （2021.08 - 至今）

角色：Java 开发工程师

项目简介：

大型 SaaS 供应链系统，提供从供应商管理、采购管理到库存管理的一体化解决方案。通过将信息流、物流、资金流进行项目设计，从采购原材料开始，到制成中间产品及最终产品，最后由销售网络把产品送到消费者手中，将供应商、制造商、分销商、零售商及最终用户连成一个整体的功能网络。目前已服务多家大型连锁餐饮企业。

技术栈：Spring Boot MySQL MyBatis Kafka Elasticsearch Redis Thrift Zebra S3 Nginx Crane DTS

主要成果：

  • 采购订单状态机 + 事件通知链路建设：基于 DTS 监听 Binlog，通过状态机将订单状态变更事件写入 MQ，构建事件日志并推送至异步业务订阅方，实现业务解耦、故障告警与重试机制，提升系统可维护性
  • 供应商绩效考核体系重构：设计「考核模型 → 考核指标 → 评分规则」的三层抽象结构，支持维度/权重的配置化动态调整，基于策略模式支持多种指标评分方式（数值/枚举/区间），通过模板引擎+数据采集+评分引擎+全流程异步化实现自动计算，保证数据一致性，支撑 5,000+ 商户级的自动评分
  • 收货单流程标准化重构：基于 OrderWizard 框架将接口方法拆分为校验/参数组装/资源调用/DB 写入等标准阶段，明确职责边界。应用策略模式承接 5 种业务场景（采购直送/统配直送/直送/越库/供应商直送），支持按需扩展。在资源调用阶段引入 Saga 补偿机制，确保库存、应付等操作最终一致性
  • 采购报价核心查询性能优化：设计「价格查询预计算模型」，从实时计算转为直接查询预计算结果，构建 SKU 维度表 + 价格结果表 + 流水表组合的协同查询架构。以商品-仓库-供应商-数量为维度，支持固定/阶梯/折扣等多种定价策略

2. 供应商协同平台 （2021.08 - 至今）

角色：Java 开发工程师

项目简介：

供应商协同平台是 SaaS 供应链系统的配套平台，为供应商提供 Web 端和微信小程序双端数字化协同入口，覆盖了供应商与商家之间的采购合同、订单管理、询价报价等全链路业务。供应商协同平台与内部商家端深度对接，实现了采购订单下达、在线接单、发货通知、采购计划共享、询价报价、对账结算等功能，打通内外部商家系统，降低供应链沟通成本，提高采购协同效率。

技术栈：Spring Boot Thrift Kafka MySQL Zebra MyBatis Elasticsearch DTS Apache POI Crane

主要成果：

  • 多租户路由框架建设：供应商注册后与商家绑定，存在一个供应商服务多个商家的场景，需要基于 RPC 调用粒度控制路由。设计并实现基于 AOP + 自定义注解 + SpEL 的轻量级多租户路由切面，支持全局/局部/方法级路由动态切换，业务代码零侵入，支撑后续灰度验证与平滑迁移
  • AI 驱动接口提效工具：梳理供应商端多场景接口与复杂对象规范，明确职责边界；利用 AI Agent Skill 实现 Thrift 接口代码自动化生成，包括接口定义生成、实体类转换、参数校验和日志记录等。将原本数小时的接口开发工作压缩至分钟级，提升团队研发效率。制定代码规范、权限控制框架，并将 Skill 推广至团队其他项目
  • 系统稳定性治理：通过限流熔断、RPC 调用超时优化、接口响应日志链路追踪、精简返回 DTO 字段、消息消费升级消费监控等方式，配合 DAS 平台定位和分析慢查询，将系统核心接口响应时间优化至 500ms 以下，有效降低集群压力
